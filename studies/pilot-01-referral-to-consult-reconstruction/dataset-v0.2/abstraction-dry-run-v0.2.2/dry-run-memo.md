# Dry-Run Memo

## 1. Why Only 2 Cases Were Abstracted

Only two cases were abstracted because Dataset v0.2.2 is a safety and workflow dry run, not the full Dataset v0.2 abstraction step. Limiting the batch makes it possible to inspect whether the protocol prevents copied source text, patient-identifying information, and clinical-guidance language before scaling.

## 2. Whether Abstraction Can Be Done Without Copying Source Narrative Text

Yes, at least in this limited dry run. The abstractions were written as high-level educational artifacts from the source-selection metadata and v0.2.1 abstraction targets. They do not reproduce source narrative text or source structure.

## 3. Whether Privacy Risk Was Manageable

Privacy risk was manageable because no demographics, exact dates, locations, institutions, record numbers, initials, images, or unusually specific sequences were retained.

## 4. Whether Copyright Risk Was Manageable

Copyright risk was manageable because the dry run used citation references only and avoided source language, figures, tables, captions, and detailed narrative sequences.

## 5. Whether the Abstractions Appear Useful for Reconstruction Coding

The abstractions appear potentially useful for reconstruction coding because each preserves a clear cognitive transition:

- ABS-001: uncertain endodontic-periodontal mechanism to mechanism-focused downstream reasoning.
- ABS-002: unresolved chest-pain uncertainty to compressed downstream risk framing.

They may be slightly too generalized, which should be evaluated during human review.

## 6. Whether a Human Reviewer Should Inspect These Before Coding

Yes. A human reviewer should inspect both abstractions before coding. The reviewer should verify copied-text distance, non-identifiability, non-guidance language, and adequate cognitive-transition specificity.

## 7. Whether the Remaining 8 Candidates Should Be Abstracted Now

No. The remaining eight candidates should not be abstracted until these two dry-run abstractions have been reviewed. If the reviewer accepts the level of abstraction and safeguards, the remaining candidates can proceed in small batches.

## Research Note 002 Timing

Research Note 002 should wait until full abstraction, two-coder coding, and reconciliation are complete. This dry run is an internal workflow test and should not be treated as an empirical result.
