# Review Decision Matrix

This matrix defines how human review outcomes control the coding gate.

## Rules

- If both abstractions are marked proceed to coding, coding may begin for these 2 only.
- If one abstraction needs revision, revise only that abstraction and repeat review.
- If either abstraction has high privacy or copyright risk, do not code it.
- If non-guidance concern remains unresolved, do not code it.
- If methodological usefulness is too low, do not code it.
- The remaining 8 candidates should not be abstracted until the dry-run review is complete.

## Interpretation Table

| Review outcome | Action |
| --- | --- |
| Both proceed to coding | Open coding gate for ABS-001 and ABS-002 only. |
| One proceeds, one needs revision | Open coding gate only for the approved abstraction if no safety concern remains; revise the other. |
| Either has high privacy risk | Keep coding gate closed for that abstraction. |
| Either has high copyright or narrative-proximity risk | Keep coding gate closed for that abstraction. |
| Either has unresolved non-guidance concern | Keep coding gate closed for that abstraction. |
| Either is too thin for coding | Keep coding gate closed and revise or replace. |
| Both need revision | Keep coding gate closed. |

## Scaling Rule

Do not abstract the remaining 8 candidates until the two-case dry-run review is complete and the review outcome has been documented.
