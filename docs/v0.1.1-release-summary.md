# v0.1.1 Release Summary

Release: `v0.1.1 — Local MCP Server + Clinical Validation Dataset`

## Why MCP Matters

MCP turns the Dental Case Packet builder into a local tool layer that AI agents can call through a standard interface. Instead of asking agents to shell out manually or parse project internals, agents can call focused tools for building packets, validating packets, summarizing packet context, listing supported formats, and checking PHI risk.

This keeps the project aligned with its role as AI-native dental data infrastructure rather than a dental chatbot or diagnostic model.

## Why Local-First Matters

Dental records may contain sensitive patient information. A local-first workflow lets dental teams and developers transform records into structured context without sending patient data to external APIs.

The MCP server follows the same safety boundary as the CLI:

- No diagnosis.
- No treatment recommendations.
- No clinical interpretation.
- No external API calls.
- Dentist-review-only outputs.

## Why Validation Datasets Matter

The Clinical Validation Dataset v0.1 provides a repeatable benchmark for infrastructure behavior. It verifies that Dental Case Packet generation, MCP execution, schema validation, summary generation, and PHI risk checks work across representative dental workflow categories.

The dataset includes 20 public-source-referenced, anonymized local fixtures covering:

- Implant planning
- Missing teeth
- Orthodontics
- Endodontics
- Wisdom teeth
- Full jaw anatomy

Validation metrics:

- 20/20 packet builds successful.
- 20/20 MCP executions successful.
- 20/20 validations successful.
- PHI risk: low=20.

## Relationship To Future Dental AI Workflows

Future Dental AI workflows need reliable context before they need model behavior. v0.1.1 strengthens the context layer by exposing local agent tools and adding a repeatable validation suite.

This release supports future work on:

- Batch MCP validation.
- Dataset provenance fields.
- Better unsupported-format handling.
- JSON Schema validation in MCP.
- DICOM thumbnail services.
- FHIR interoperability.
- Dental RAG and agent SDK patterns.

The project remains AI-native Dental Data Infrastructure and a Local-first Dental Context Layer for AI Agents. It is not a diagnostic system, treatment recommendation system, medical device, or clinical decision system.
